# Telegram CRM Bot  

**Telegram CRM Bot** — это корпоративная CRM‑система, реализованная в формате Telegram‑бота и предназначенная для автоматизации бизнес‑процессов в малом и среднем бизнесе.  

Проект разработан в рамках финальной работы курса **«Основы Python. Часть 2»** и демонстрирует навыки backend‑разработки, проектирования реляционных баз данных, построения архитектуры серверных приложений, а также интеграции Telegram‑интерфейса с бизнес‑логикой.

Система включает управление клиентами и сделками, постановку задач, автоматические уведомления, аналитику, журналирование действий пользователей и интеллектуальные рекомендации на основе локальной LLM‑модели.

## Описание проекта
Система представляет собой **полнофункциональную CRM‑платформу**, доступную через Telegram‑бота.  

Функциональные возможности включают:

- управление клиентской базой;
- ведение сделок и этапов продаж;
- постановку и контроль задач сотрудников;
- автоматические уведомления и напоминания;
- формирование аналитических отчётов;
- журналирование действий пользователей (аудит);
- интеллектуальные рекомендации на основе локальной LLM‑модели.

Проект демонстрирует навыки разработки корпоративных систем, построения архитектуры, работы с ORM, проектирования БД и интеграции внешних сервисов.

---

## Архитектура системы
Система реализована на основе **клиент‑серверной архитектуры**, где Telegram выступает в роли пользовательского интерфейса, а серверная часть обрабатывает бизнес‑логику.

### Основные компоненты:

- **Telegram Bot (aiogram)** — интерфейс взаимодействия с пользователями;
- **Backend (Python)** — обработка запросов, логика CRM, аналитика;
- **MySQL** — централизованное хранилище данных;
- **SQLAlchemy ORM** — слой взаимодействия с БД;
- **APScheduler** — планировщик фоновых задач;
- **Локальная LLM‑модель (TinyLLaMA)** — интеллектуальная аналитика и рекомендации.

## Структура базы данных
База данных MySQL включает следующие сущности:

- `users` — пользователи  
- `clients` — клиенты  
- `deals` — сделки  
- `tasks` — задачи  
- `interactions` — история взаимодействий  
- `notifications` — уведомления  
- `mailings` — рассылки  
- `reports` — отчёты  
- `audit_logs` — журнал действий  

Реализованы:

- внешние ключи;
- индексация;
- ограничения целостности;
- асинхронный доступ;

# ⚙ Инструкция по установке и запуску проекта

Данный раздел содержит **пошаговую инструкцию по установке, настройке и первому запуску Telegram CRM Bot**.

Следуя этим шагам, вы сможете развернуть проект локально и запустить  Telegram-бота.

---

## 1️⃣ Клонирование репозитория

Склонируйте проект с GitHub и перейдите в каталог проекта:

```bash
git clone https://github.com/<username>/Pollmoriy.git
cd Pollmoriy
```

## 2️⃣ Создание виртуального окружения
Создайте виртуальное окружение Python:
```bash
python -m venv venv
```
Активируйте его:
```
Windows
venv\Scripts\activate
Linux / macOS
source venv/bin/activate
```
## 3️⃣ Установка зависимостей
Установите все необходимые библиотеки:
```
pip install -r requirements.txt
```
## 4️⃣ Настройка переменных окружения
В корне проекта создайте файл .env и заполните его следующим образом:
```
BOT_TOKEN=your_telegram_bot_token

DB_NAME=business_manager
DB_USER=root
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=3306
```
⚠️ Файл .env не добавляется в репозиторий и должен находиться в .gitignore.
Он содержит конфиденциальные данные и не должен публиковаться.

## 5️⃣ Создание базы данных MySQL
Создайте новую базу данных в MySQL:
```
CREATE DATABASE business_manager
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```
## 6️⃣ Инициализация таблиц
При первом запуске проекта все таблицы создаются автоматически:
```
python main.py
```
В консоли появится сообщение:
```
База данных готова!
```
## 7️⃣ Запуск Telegram-бота
Для запуска бота выполните:
```
python main.py
```
После успешного запуска бот станет доступен в Telegram.

## 8️⃣ Первый вход в систему
Первый пользователь, отправивший боту команду:

/start
➡ автоматически регистрируется как администратор системы и получает полный доступ ко всем функциям.

## 9️⃣ Типовые ошибки и способы их решения
❌ Ошибка подключения к базе данных
Проверьте:

- запущен ли MySQL-сервер;

- корректность параметров подключения в .env;

- существует ли база данных business_manager;

- открыт ли порт 3306.

❌ Бот не запускается
Проверьте:

- активировано ли виртуальное окружение;

- установлены ли все зависимости (pip install -r requirements.txt);

- корректен ли BOT_TOKEN.

❌ Ошибки импорта модулей
Выполните повторную установку зависимостей:
```
pip install --upgrade -r requirements.txt
```
